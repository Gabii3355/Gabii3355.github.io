FASTQ read filtering and trimming in R

This project presents a small bioinformatics workflow for analysing sequencing read quality, filtering low-quality reads and trimming FASTQ sequences using GNU R and the ShortRead package.

View results Open R script Open report

Project overview

Goal

The aim was to inspect FASTQ read quality, identify problematic reads and apply filtering and trimming rules to improve the final dataset.

Input data

FASTQ files containing DNA reads and base quality scores encoded as Phred values.

Tools

GNU R, R Markdown, ShortRead, Biostrings, base R visualisation.

Workflow

1

Load FASTQ file with ShortRead

2

Extract sequences and quality scores

3

Convert encoded quality values to a numeric Phred matrix

4

Generate quality boxplots and inspect low-quality regions

5

Filter reads and trim sequence ends

6

Save processed FASTQ file and final quality plot

Quality control results

The original reads were evaluated using per-base Phred quality scores. Low-quality positions were visible as lower values in the boxplot, especially near read ends. Based on this, the reads were filtered and trimmed to obtain a cleaner dataset.

Filtering and trimming strategy

Reads were removed when their quality profile indicated unreliable sequencing results. The remaining reads were trimmed to reduce low-quality sequence regions.

  • Quality values were analysed as Phred scores.
  • Reads with too many bases below the quality threshold were removed.
  • Read ends were trimmed to keep the more reliable part of the sequence.
  • The processed reads were saved as a new FASTQ file.

Example code fragment

library(ShortRead)

fastq <- readFastq("example.fastq")
seq <- sread(fastq)
q <- quality(fastq)

qual <- as(q, "matrix")

avgQ <- rowMeans(qual)
lowQ <- rowSums(qual < 20)

keep <- (avgQ > 25) & (lowQ < 10)

filtered <- fastq[keep]
trimmed <- narrow(filtered, start = 1, end = 100)

writeFastq(trimmed, "example_filtered_trimmed.fastq", compress = FALSE)
