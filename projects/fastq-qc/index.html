Project 03 · Bioinformatics mini project

FASTQ Quality Control in R

A bioinformatics workflow for analysing sequencing read quality, filtering low-quality reads and trimming FASTQ data using GNU R, R Markdown and Bioconductor.

The project focuses on reproducible preprocessing of sequencing reads: quality inspection, Phred score analysis, read filtering, trimming and export of a cleaned FASTQ file.

FASTQ R R Markdown ShortRead Phred quality

Project goal

How can low-quality FASTQ reads be detected and improved before downstream analysis?

The aim of this project was to inspect read quality, identify problematic regions in sequencing data and apply filtering and trimming rules to obtain a cleaner FASTQ dataset.

01

Inspect read quality

Convert encoded quality scores into numeric Phred values and visualize quality across read positions.

02

Filter unreliable reads

Remove reads that contain too many low-quality bases or do not meet selected quality criteria.

03

Trim low-quality ends

Shorten reads to remove unreliable sequence regions while preserving the link between bases and quality scores.

Workflow

  1. Load the FASTQ file using the ShortRead package.
  2. Extract DNA sequences and quality scores from the FASTQ object.
  3. Convert encoded quality values into a numeric Phred quality matrix.
  4. Generate quality boxplots before processing.
  5. Apply filtering rules to remove unreliable reads.
  6. Trim selected read regions based on quality profile.
  7. Save the processed FASTQ file and final quality plot.

Results

Quality plots before and after processing

The original reads were evaluated using per-base Phred quality scores. After filtering and trimming, the final dataset retained reads with a more reliable quality profile.

Filtering decision

How the reads were improved

Decision 1

Low-quality reads were removed.

Reads containing too many bases below the selected Phred quality threshold were excluded to reduce unreliable sequence information.

Decision 2

Read ends were trimmed.

The quality profile showed that read ends can contain less reliable positions, so trimming was used to keep the more trustworthy part of each read.

Decision 3

Sequences and quality scores were processed together.

Filtering and trimming were performed on the FASTQ object to preserve the connection between DNA bases, quality values and read metadata.

Code example

Core R workflow

The analysis was prepared in R Markdown and exported as a standalone R script. The script can process FASTQ input and save a cleaned FASTQ file together with a final quality plot. 

library(ShortRead)

fastq <- readFastq("input.fastq")
seq <- sread(fastq)
q <- quality(fastq)

qual <- as(q, "matrix")

low_quality_count <- rowSums(qual < 20)
keep <- low_quality_count < 10

filtered <- fastq[keep]
trimmed <- narrow(filtered, start = 1, end = 100)

writeFastq(trimmed, "processed_reads.fastq", compress = FALSE)
